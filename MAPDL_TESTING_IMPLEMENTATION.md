# MAPDL Code Generation Testing Implementation

This document describes the testing framework implemented for validating MAPDL code generated by an LLM when using the `PYMAPDL_SYSTEM_PROMPT` from the pymapdl-mcp MCP server.

## What Was Implemented

### Phase 1: Foundation & Architecture ✅

#### 1. Dependencies Added to `pyproject.toml`
- `genai-test` - For tracking LLM responses and exchanges
- `openai` - For Azure OpenAI integration
- `pytest-vcr` - For recording/replaying LLM responses
- Updated pytest markers: `llm_integration` and `llm`

#### 2. Test Utilities Module (`tests/test_utils.py`)
Provides core utility functions for code extraction and adaptation:

- **`extract_executable_code(response: str) -> str`**
  - Extracts Python code from various LLM response formats
  - Handles markdown code blocks, plain code blocks, and pattern matching
  - Validates extracted code using AST parsing
  - Robust error handling with clear diagnostic messages

- **`adapt_code_for_execution(code: str) -> str`**
  - Prepares generated code for safe test execution
  - Comments out MAPDL launch/exit commands
  - Removes ansys package imports
  - Preserves actual MAPDL operations

- **`get_llm_context() -> tuple[str, list]`**
  - Returns system prompt and tool descriptions
  - Falls back gracefully if ansys.mapdl.mcp not installed
  - Ensures tests use actual production prompt

#### 3. Extended `tests/conftest.py`
Added new fixtures for LLM integration testing:

- **`stateful_mock_mapdl`** - A mock MAPDL instance that:
  - Tracks entity creation (keypoints, lines, areas, volumes)
  - Updates entity counts dynamically
  - Executes MAPDL commands without errors
  - Allows tests to run without MAPDL license

- **`azure_openai_client`** - Azure OpenAI client fixture that:
  - Validates credentials via environment variables
  - Gracefully skips tests if credentials not configured
  - Initializes with proper API version

### Phase 2: Base Template Class ✅

**File: `tests/test_mapdl_code_generation.py`**

`TemplateTest` - Reusable base class for LLM code generation tests with:

#### Configuration Attributes
- Entity counts: `num_nodes`, `num_elements`, `num_keypoints`, `num_lines`, `num_areas`, `num_volumes`
- Specific entities: `keypoints`, `lines`, `areas`, `volumes`, `nodes`, `elements`
- Code validation: `commands`, `words`, `parameters`, `functions`
- Query: `query` - The task for the LLM

#### Lifecycle Methods
- `run_llm_query()` - Query LLM for MAPDL code
- `extract_code()` - Extract Python from response
- `adapt_code()` - Clean code for test execution
- `execute_code()` - Run code against MAPDL

#### Built-in Test Methods (auto-discovered by pytest)
- `test_code_execution_succeeds()` - Code runs without errors
- `test_entity_count()` - Correct number of entities created
- `test_command_usage()` - Expected PyMAPDL methods used
- `test_response_contains_words()` - Words in LLM response
- `test_parameters()` - Parameter values match
- `test_custom_functions()` - Custom validators pass

## Usage Example

```python
from test_mapdl_code_generation import TemplateTest
from test_utils import get_llm_context
import pytest

@pytest.mark.llm_integration
class TestMyMAPDLTask(TemplateTest):
    query = "Create a box from (0,0,0) to (1,1,1)"
    num_volumes = 1
    commands = ["v"]

    @pytest.fixture(autouse=True)
    def setup_and_execute(self, mapdl_fixture, llm_client):
        self.setup_class()
        mapdl_context = get_llm_context()
        self.run_llm_query(llm_client, mapdl_context)
        self.extract_code()
        self.adapt_code()
        self.execute_code(mapdl_fixture)
        yield mapdl_fixture
```

Running tests:
```bash
# Set up Azure OpenAI credentials
export AZURE_OPENAI_KEY="..."
export AZURE_OPENAI_ENDPOINT="..."
export AZURE_OPENAI_DEPLOYMENT="..."

# Run LLM integration tests
pytest tests/test_mapdl_code_generation_examples.py -v -m llm_integration

# Or with mock MAPDL (no license needed)
# Tests will automatically use mock by default
pytest tests/test_mapdl_code_generation_examples.py -v -m llm_integration

# Or with real MAPDL
export USE_REAL_MAPDL="true"
pytest tests/test_mapdl_code_generation_examples.py -v -m llm_integration
```

## Next Steps

To add your own tests:

1. **Create a new test class** inheriting from `TemplateTest`
2. **Define the task** by setting the `query` attribute
3. **Specify expected outcomes** (entity counts, commands, etc.)
4. **Implement the fixture** to orchestrate execution
5. **Run the tests** - validation methods are auto-generated

See `tests/MAPDL_CODE_GENERATION_TESTING.md` for detailed guidance.
